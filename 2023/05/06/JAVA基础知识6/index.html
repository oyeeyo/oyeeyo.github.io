JAVA学习笔记6

1.方法的定义及好处

JAVA的方法的用法类似于C语言的函数,方法是程序中最小的执行单元,例如

方法的用处:方法提高了代码的复用性,可以重复调用，避免代码的冗长，以及提高了代码的可维护性，将复杂的代码分成若干个代码块，便于后续人员对代码进行维护和修改

2.方法的定义及调用

方法的定义只有一种形式,可以类比于C语言函数的定义,但方法需要在类里面进行定义才有效,否则可能会报错(至少定义的是一个无效的方法 ︿(￣︶￣)︿),举个小小的例子:

接下来学习一下方法的另外的两种用法:带参数的方法和带返回值的方法.这两种方法其实跟C语言的函数的用法相差不大(毕竟JAVA的底层语言就是使用C语言嘛!).直接举个例子吧

3.方法的小结

  1. 方法不调用就不会执行，当然不调用程序也不会出错
  2. 方法与方法之间是平级关系,不能够互相嵌套定义. 就比如将主方法与定义的其他方法互换位置(其他方法定义在主方法前面)后依然能够在主方法中正常调用其他方法。注:这一点与C语言是不一样的,C语言如果要调用函数，那么至少需要在主函数前面声明一下这个函数
  3. 方法的执行顺序与编写顺序无关,与在主方法中的调用顺序有关
  4. 方法的返回值类型为void时表示该方法没有返回值,则可以省略return语句不写.如果要编写return,后面不能跟具体的数据(return 语句表示该方法结束,返回主方法,即:在return后面编写的语句均为无效语句)

方法的重载:在同一个类当中,方法名相同，参数相同的方法。(与返回值无关)而参数的不同又分为:参数的个数不同，类型不同，顺序不同

4.方法的基本内存原理

其实方法运行时使用的内存是在栈空间中进行的,方法进栈运行，运行完毕就出栈，而对于new出来的则是由堆空间分配的,比如new的数组(引用数据类型的变量实际上存储的是地址值),另外如果方法传入的参数为引用数据类型,实际上传入的是其对应的地址值
